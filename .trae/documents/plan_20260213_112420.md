Berikut adalah hasil analisis mendalam dan rencana implementasi untuk fitur "Override Mode" serta perbaikan logika sistem.

### 1. Analisis & Laporan Audit Logika

Saya menemukan **inkonsistensi kritikal** dalam arsitektur sistem saat ini:

*   **Duplikasi & Bypass Logika (Major Issue):**
    *   **Logic Pusat:** `AttendanceService::checkIn` memiliki logika lengkap (validasi jadwal, cek cuti/izin otomatis, penalti keterlambatan).
    *   **Logic Frontend:** Komponen Livewire `CheckInOut.php` **tidak menggunakan service tersebut**. Ia menulis langsung ke database (`Attendance::create(...)` pada baris 180).
    *   **Dampak:** Fitur penting seperti "Cek Cuti Otomatis" (yang mengubah status jadi 'excused') **tidak berjalan** jika user check-in lewat halaman ini. Logika penalti juga terduplikasi.

*   **Ketergantungan Jadwal:**
    *   Saat ini, sistem **wajib** memiliki `ScheduleAssignment` untuk membuat absensi. Kolom `schedule_assignment_id` di database sudah `nullable` (bagus), tapi logika kode masih memblokir jika tidak ada jadwal.

### 2. Rencana Implementasi "Override Mode"

Saya mengusulkan perbaikan arsitektur sekaligus implementasi fitur override:

#### Tahap 1: Konfigurasi & Database
1.  Menambahkan setting `attendance.override_mode` (default: `true` sesuai request) di `config/siwirus.php`.
2.  Memastikan database `attendances` kolom `schedule_assignment_id` benar-benar nullable (sudah terverifikasi aman).

#### Tahap 2: Refactoring `AttendanceService` (Central Logic)
1.  Memodifikasi method `checkIn` agar parameter `$scheduleAssignmentId` menjadi opsional (`?int`).
2.  Jika `$scheduleAssignmentId` kosong (null):
    *   Lewati validasi kepemilikan jadwal dan jam jadwal.
    *   Set status default menjadi `'present'` (Hadir).
    *   Lewati logika penalti keterlambatan.

#### Tahap 3: Refactoring Livewire `CheckInOut` (UI Logic)
1.  **Hapus logika duplikat:** Ganti `Attendance::create(...)` dengan memanggil `AttendanceService::checkIn(...)`.
2.  **Logika Override:**
    *   Ubah validasi `checkIn()`: Jika tidak ada jadwal aktif, cek apakah `override_mode` aktif.
    *   Jika aktif, izinkan lanjut dan kirim `null` sebagai ID jadwal ke Service.
    *   Tampilkan tombol Check-in di UI meskipun tidak ada jadwal (jika mode override aktif).

### 3. Perbaikan Bug Lainnya
